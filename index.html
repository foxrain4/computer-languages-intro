Understanding Computer Languages

Overview

Computer languages are the tools we use to speak to machines. They shape how websites, games, and apps are created—from logic to visuals. Understanding the basics can help you better plan, communicate, and even prototype ideas for your game world or creative projects.

Why It Matters

Whether you're building a card game, creating an interactive map, or tweaking an online interface, knowing what each language does can help you work smarter or collaborate better with developers. Even a basic grasp can unlock new creative control.

Core Concepts

  • HTML (Structure): The skeleton of a webpage. It defines headings, sections, images, and text.
  • CSS (Style): Controls how things look—colors, fonts, layout, animations.
  • JavaScript (Behavior): Adds interactivity, such as buttons, filters, and dynamic effects.
  • Markdown: A simple shorthand for writing text with basic formatting. Often used for documentation.
  • YAML & JSON: Lightweight data languages for settings, configs, or card/game data.

Examples in Games

Some worldbuilding sites use HTML and CSS to create interactive lore maps. Game designers often use JSON or YAML to store card properties, spells, or unit stats. Even Markdown plays a role in writing readable documentation for game systems.

How to Apply It

  • Try writing a simple HTML page that displays your world lore or factions.
  • Use JSON to structure a sample card's stats: name, cost, ability.
  • Ask yourself: “How can I present this story element visually?”
  • Test Markdown in your wiki or docs to keep it simple and clean.
